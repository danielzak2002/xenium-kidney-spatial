Xenium kidney pilot — Phase-A summary

Dataset: kidney_preview_PRCC · modality: gene only · 56504 cells · generated 2026-06-05
Data: 10x Genomics Xenium human kidney (CC BY 4.0). Code: MIT.

Scope. Validation of the parametrized R/Seurat loop (load → QC → normalize → cluster → annotate → spatial smoke-test) on the smaller of the two datasets, the small→big gate target. The identical code (R/config.R, 01, 02) re-points at the full multimodal section via XENIUM_DATASET=big.

Pipeline steps

  1. Panel assessment — parse gene_panel.json + cell_feature_matrix.h5; confirm immune/B-cell & plasma markers; enumerate control features.
  2. Load & QC (01_load_qc.R) — lean LoadXenium (centroids only); Xenium-specific QC from cells.parquet: flag (not filter) negative-control / blank-codeword / segmentation-merge cells; remove only zero-count cells.
  3. Normalize & cluster & annotate (02_cluster_annotate.R) — LogNormalize (all panel genes as features), PCA/UMAP, native igraph Leiden, marker DE, provisional marker-score annotation.
  4. Spatial smoke-test — immune-neighbourhood fraction on cell centroids (full spatial statistics are Phase B / squidpy).

Conclusions & observations

The loop is validated. QC reproduces the vendor metrics exactly (median 57 transcripts / 39 genes per cell), and clustering is biologically coherent on the first pass.
  • Reference annotation refines the provisional labels, and a marker lineage gate resolves the cross-method conflicts (CD3 for T vs non-T, PTPRC for immune vs epithelial) — recording the evidence and never forcing ambiguous calls (see the reference section). These are the labels to review.
  • Biology tracks the known sample. This papillary renal cell carcinoma (PRCC) section recovers coherent compartments, named here by each cluster's own top markers: tumour (MET, CD70, C15orf48, ANPEP, EGFR); myeloid (MARCO, LILRB4, PLA2G7, VSIG4, CD68); endothelium (VWF, EGFL7, ANGPT2, CD34, ADGRL4); T cell (LAG3, TNFRSF9, CD8A, GZMK, GZMA); B cell (MS4A1, BANK1, CD19, TNFRSF13B, CD79A); plasma (CD79A, MZB1, TENT5C, DERL3, FKBP11). Immune lineages account for ~61% of provisional labels — consistent with the dataset's immune-rich, infiltrated description.
  • B-cell / plasma biology is resolvable despite CD79B being absent from the panel: distinct B-cell (MS4A1, BANK1, CD19, TNFRSF13B, CD79A) and plasma (CD79A, MZB1, TENT5C, DERL3, FKBP11) clusters separate cleanly and localise to discrete foci (B/plasma-signal panel) — a promising substrate for the immune-aggregate / TLS question.
  • The immune-heavy composition is largely real, not only a scorer artefact. The section is a tumour with extensive infiltrate; epithelial (nephron) representation is correspondingly sparse, which also makes tubular cell-typing the weakest part of the provisional (z-scored marker-argmax) labels.
  • QC flags are low (mean neg-control fraction 0.00027; 109 candidate segmentation merges of 56504 cells) — segmentation looks trustworthy for cell-level analysis.
Caveats & next steps.
  • Cell-type labels are provisional (marker argmax). Replace with reference-based label transfer (e.g. a kidney/immune atlas) before any quantitative cell-type claim.
  • The spatial smoke-test is a sanity check only; neighbourhood enrichment, co-occurrence, Moran's I and niche detection are done in Phase B (squidpy / spatialdata).
  • This is the smaller validation section; the full multimodal section (XENIUM_DATASET=big) adds the 27-plex protein and is loaded via the manual h5 path that handles the panel's Deprecated Codeword features.
